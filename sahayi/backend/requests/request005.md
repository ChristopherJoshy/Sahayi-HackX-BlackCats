# Request 005 — Switch vision from OpenAI to cheap Gemini

## Change
- `voice/vision.py`: prescription image extraction now uses Gemini vision
  (`google-genai` SDK) instead of OpenAI. Default model `gemini-2.0-flash-lite`
  (cheapest Gemini vision tier).
- `core/config.py`: replaced `openai_api_key`/`openai_model` with
  `gemini_vision_api_key`/`gemini_vision_model` (env `GEMINI_VISION_API_KEY`,
  `GEMINI_VISION_MODEL`, default `gemini-2.0-flash-lite`).
- `agents/whatsapp_agent.py`: image messages now run Gemini vision to extract
  prescription text, which is folded into the prompt; the Malayalam reply is still
  generated by Sarvam. Removed the OpenAI vision client.
- `core/bootstrap.py`: health service `openai` → `gemini_vision`.
- `requirements.txt`: re-added `google-genai`.
- `.env.example` + README updated (Gemini vision only; no OpenAI).

## Rationale
Gemini `gemini-2.0-flash-lite` is the cheapest multimodal option for prescription
OCR; Sarvam remains the sole text LLM + voice provider.

  `agents/whatsapp_agent.py`, `voice/vision.py`: OK.
- No lingering OpenAI vision references (only the Sarvam-via-OpenAI-SDK client in
